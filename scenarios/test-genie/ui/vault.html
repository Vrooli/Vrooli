Test Vaults

Vaults orchestrate an end-to-end validation journey for a scenario. Instead of kicking off individual suites manually, you define the phases, articulate success criteria, and let Test Genie enforce the entire run every time.

When to use a vault

  • Pre-release sign-off that must cover setup → develop → test → deploy.
  • Customer or environment-specific smoke workflows you need on repeat.
  • Chaos or incident drills where every step must complete without intervention.

What you configure

  • Scenario & vault name for traceability.
  • Phases to run, with per-phase notes and time budgets.
  • Success criteria such as coverage thresholds or failure tolerance.

What you get back

An executable asset you can trigger from the UI or API. Test Genie stores history, streams live phase progress, and blocks completion until every requirement is satisfied.
